# Roadmap — этапы под Claude Code

Правила работы: один этап = одна-две сессии Claude Code. Перед стартом этапа дай Claude Code прочитать `CLAUDE.md` + релевантный раздел доков. Этап закрыт только когда выполнены **критерии приёмки** и тесты зелёные. Коммит после каждого работающего инкремента.

**Стратегия v1 (обновлено дважды):** сначала доводим до полностью рабочего и **задеплоенного** состояния голый бот — daily-пазл, ничего больше. Никаких новых фич (челленджи, Mini App), пока не отточено то, как база работает и ощущается в проде на живых людях. Фичи подключаем по одной, после того как ядро стабильно.

---

## Этап 0 — Скелет репозитория ✅ готово

Monorepo: `backend/` (FastAPI + aiogram, uv, ruff, pytest), `webapp/` (Vite React TS Tailwind, заглушка — не развивается, см. «Потом» ниже), `generator/` (офлайн-скрипты), `deploy/`, `.github/workflows`.
**Приёмка:** `docker compose up` локально поднимает API, `curl /api/health` → 200; `npm run dev` открывает заглушку.

## Этап 1 — Данные ✅ готово

По `DATA_PIPELINE.md`. `game_static.db` на 100 дней RU+EN; guess-словари (RU 25 673 / EN 31 914 слов) и answer-словари (RU 1593 / EN 846) прошли автоматический smoke-гейт §5 (0 нарушений) и ручную/агентскую курацию в несколько итераций.
**Приёмка:** ✅ пройдена — топ-30 контрольных слов проверен глазами, чисто.

## Этап 2 — Игровой движок + API ✅ готово

По `TECH_SPEC.md` §3–5: схема БД, ATTACH статической базы, лемматизация RU/EN (surface-first, с оверрайдами для омографов вроде «пар»/«деньги»), сервис `guess()`/`state()`, эндпоинты `/api/state`, `/api/guess`, `/api/lang`, `/api/stats`, валидация initData, rate limit.
**Приёмка:** ✅ 28 тестов зелёных, ручной curl-прогон против реальной `game_static.db` подтвердил все сценарии (новое/повтор/не найдено/победа/стрик).

## Этап 3 — Бот MVP ✅ готово

По `TECH_SPEC.md` §6: `/start`, onboarding, игровое сообщение с `editMessageText`, топ-5, `/lang`, `/stats`, `/mute`, обработка ошибок словаря (авто-удаление через 3с).
**Приёмка:** ✅ живой прогон на `@teplee_game_bot` — 14 апдейтов, 0 ошибок, игровое сообщение редактируется на месте, ранги корректны. (Прогон был локальный dev-процесс, не прод — отсюда важность Этапа 4.)

---

## Этап 4 — Деплой бота в прод · 1–2 дня

По `DEPLOY.md` Фаза 1: backend-контейнер (FastAPI + aiogram, long polling) на VPS через Docker Compose. **Без домена и без Caddy** — публичный HTTPS не нужен, пока нет Mini App и челленджей через API. GitHub Actions: test → build → push GHCR → SSH deploy. Перенос данных `push_data.sh`. apscheduler-пуш 09:00 МСК. `/api/health` мониторится локально на VPS.
**Приёмка:** бот отвечает в Telegram 24/7 без локального dev-процесса на твоей машине; push в main обновляет прод; бэкапы `game.db` работают.

## Этап 5 — Полировка + мягкий релиз голого бота · 2–3 дня

Только то, что касается ощущения от **daily-пазла как он есть**, без новых фич: тексты онбординга, `/help`, чистовая обработка edge-кейсов (сообщение удалено юзером, `message is not modified`, юзер заблокировал бота), скорость и стабильность ответа. Обкатка на 10–20 знакомых → фикс болячек.
**Приёмка:** несколько дней без крашей на живых пользователях; ощущается быстро и предсказуемо; метрики §7 PRD, которые применимы без челленджей (DAU, retention, доля дошедших до победы, медиана попыток), считаются простыми SQL-запросами или `/admin`-командой.

## Потом (после того, как голый бот отточен и стабилен в проде)

Ничего из этого не начинать, пока Этапы 4–5 не закрыты и по боту нет живых данных хотя бы за несколько дней. Порядок внутри блока — обсуждается отдельно, когда до него дойдём:

- **Челленджи «Загадай другу»** — `POST /api/challenge`, deep link `?start=c_<id>`, флоу в боте текстом (создание/пересылка/уведомление/«загадать в ответ»). По `TECH_SPEC.md` §4, §6.
- **Mini App** — тема, лента, инпут, haptics, конфетти, share, деплой веб-аппа (домен + Caddy/Cloudflare Pages появляются только тут). По `TECH_SPEC.md` §7 и `DEPLOY.md` Фаза 2.
- **Финальная полировка + широкий релиз** — с учётом того, какие фичи из списка выше уже подключены: стрик-огонёк, экран победы, пост на VC/Habr/Reddit r/telegram.

---

**Итого:** до продового голого бота — несколько дней (Этапы 4–5). Дальше не расписываем заранее — решаем по одной фиче за раз, глядя на то, как ведёт себя уже живой продукт.
