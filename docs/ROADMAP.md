# Roadmap — этапы под Claude Code

Правила работы: один этап = одна-две сессии Claude Code. Перед стартом этапа дай Claude Code прочитать `CLAUDE.md` + релевантный раздел доков. Этап закрыт только когда выполнены **критерии приёмки** и тесты зелёные. Коммит после каждого работающего инкремента.

**Стратегия v1 (обновлено):** сначала доводим до полностью рабочего и задеплоенного состояния **бот-версию** (текст + челленджи в чате, реальные игроки). Mini App — отдельный цикл **после** того, как бот живёт в проде и по нему есть первые метрики. Так продукт получает реальных пользователей и обратную связь раньше, а фронтенд строится поверх уже проверенной механики, а не параллельно с ней.

---

## Этап 0 — Скелет репозитория ✅ готово

Monorepo: `backend/` (FastAPI + aiogram, uv, ruff, pytest), `webapp/` (Vite React TS Tailwind, заглушка — не развивается до Этапа 7), `generator/` (офлайн-скрипты), `deploy/`, `.github/workflows`.
**Приёмка:** `docker compose up` локально поднимает API, `curl /api/health` → 200; `npm run dev` открывает заглушку.

## Этап 1 — Данные ✅ готово

По `DATA_PIPELINE.md`. `game_static.db` на 100 дней RU+EN; guess-словари (RU 25 673 / EN 31 914 слов) и answer-словари (RU 1593 / EN 846) прошли автоматический smoke-гейт §5 (0 нарушений) и ручную/агентскую курацию в несколько итераций.
**Приёмка:** ✅ пройдена — топ-30 контрольных слов проверен глазами, чисто.

## Этап 2 — Игровой движок + API ✅ готово

По `TECH_SPEC.md` §3–5: схема БД, ATTACH статической базы, лемматизация RU/EN (surface-first, с оверрайдами для омографов вроде «пар»/«деньги»), сервис `guess()`/`state()`, эндпоинты `/api/state`, `/api/guess`, `/api/lang`, `/api/stats`, валидация initData, rate limit.
**Приёмка:** ✅ 28 тестов зелёных, ручной curl-прогон против реальной `game_static.db` подтвердил все сценарии (новое/повтор/не найдено/победа/стрик).

## Этап 3 — Бот MVP ✅ готово

По `TECH_SPEC.md` §6: `/start`, onboarding, игровое сообщение с `editMessageText`, топ-5, `/lang`, `/stats`, `/mute`, обработка ошибок словаря (авто-удаление через 3с).
**Приёмка:** ✅ живой прогон на `@teplee_game_bot` — 14 апдейтов, 0 ошибок, игровое сообщение редактируется на месте, ранги корректны.

## Этап 4 — Челленджи «Загадай другу» (бот-only) · 2–3 дня

По `TECH_SPEC.md` §4 (`POST /api/challenge`) и §6: расчёт рангов по матрице векторов, deep link `?start=c_<id>`, весь флоу — **только в боте** (создание слова текстом/командой, ссылка, пересылка, прохождение, уведомление создателю, «загадать в ответ»). Таблица результатов — текстом в чате (`/mychallenges` или аналог), без Mini App. `?startapp=c_*` и мини-таблица в интерфейсе — переносятся в Этап 7 вместе с Mini App.
**Приёмка:** полный цикл на двух аккаунтах (ты + брат): создал → переслал → друг угадал → тебе пришло уведомление → «загадать в ответ». Время создания челленджа ≤ 3 сек.

## Этап 5 — Деплой бота в прод · 1–2 дня

По `DEPLOY.md` (обновлён): backend-контейнер (FastAPI + aiogram, long polling) на VPS через Docker Compose. **Без домена и без Caddy** — публичный HTTPS для API не нужен, пока нет Mini App. GitHub Actions: test → build → push GHCR → SSH deploy (только `backend/**`). Перенос данных `push_data.sh`. apscheduler-пуш 09:00 МСК. `/api/health` мониторится локально на VPS (curl из cron или существующего алерт-бота через SSH-туннель/внутреннюю сеть — без публичной экспозиции).
**Приёмка:** push в main → прод обновлён; бот отвечает в Telegram 24/7; ежедневный пуш приходит; бэкапы `game.db` работают.

## Этап 6 — Полировка + мягкий релиз бот-версии · 2–3 дня

Тексты онбординга, `/help`, красивый username бота, аватар, описание, чистовая обработка edge-кейсов (сообщение удалено юзером, `message is not modified`, юзер заблокировал бота). Обкатка на 10–20 знакомых → фикс болячек → мягкий анонс (без большого паблика — это делаем после Mini App).
**Приёмка:** 3 дня без крашей на живых пользователях; метрики §7 PRD считаются простыми SQL-запросами или `/admin`-командой в самом боте (DAU, новые, победы, стрики).

## v1.5 — Mini App (отдельный цикл, после метрик по боту)

Не стартует, пока Этапы 4–6 не закрыты и по боту нет хотя бы недели живых данных. Тогда:

### Этап 7 — Mini App
По `TECH_SPEC.md` §7: тема, лента, инпут, цвета/градиент, haptics, победа с конфетти, share, синхронизация с ботом (`/api/state`). Возвращаем и включаем кнопку «🎮 Открыть игру» в игровом сообщении (сейчас скрыта — см. Этап 4/5 код).
**Приёмка:** слово, введённое в чате бота, видно в Mini App и наоборот; тема тёмная/светлая корректна; haptics работают на телефоне.

### Этап 8 — Деплой веб-аппа
По `DEPLOY.md`: теперь нужен домен + Caddy (или Cloudflare Tunnel) для публичного HTTPS API, плюс Cloudflare Pages для статики. `?startapp=c_*` deep link, мини-таблица результатов челленджа в интерфейсе.
**Приёмка:** Mini App открывается с прода по HTTPS; deep link челленджа открывает игру сразу в нужном режиме.

### Этап 9 — Финальная полировка + широкий релиз
Shake-анимация, стрик-огонёк, экран победы с конфетти. Пост на VC/Habr/Reddit r/telegram + тематические чаты.
**Приёмка:** метрики §7 PRD (включая K-factor и конверсию share→игра) считаются по обоим каналам (бот + Mini App).

---

**Итого v1 (бот):** ~1–1.5 недели до продовой бот-версии с челленджами. Mini App и широкий релиз — отдельный цикл (~2 недели) после того, как бот покажет реальные метрики.
