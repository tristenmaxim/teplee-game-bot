# Data Pipeline — словари и ранги (офлайн, локальный ПК)

Самый важный этап: словарь — душа игры. Всё выполняется в папке `generator/` на локальной машине. Результат — артефакты, которые кладутся на сервер:

| Артефакт | Что это |
|---|---|
| `game_static.db` | Таблицы `words_rank`, `daily_words` на N дней вперёд |
| `vectors_ru.npy` + `vocab_ru.txt` | Матрица векторов guess-словаря RU (для челленджей) |
| `vectors_en.npy` + `vocab_en.txt` | То же для EN |
| `en_forms.json` | Словарь форм EN → лемма (для лемматизации на сервере) |

## 0. Модели эмбеддингов

- **RU:** `navec_hudlit_v1_12B_500K_300d_100q.tar` (Navec от проекта Natasha, ~50 MB, качество хорошее).
- **EN:** fastText `crawl-300d-2M.vec` (или GloVe 840B). Скачивается один раз, ~4–7 GB на диске — только локально.

## 1. Два словаря — две роли

1. **Guess-словарь (~30–40k слов/язык):** всё, что игрок может вводить и получать ранг. Из него же строится матрица векторов и все ранги.
2. **Answer-словарь (~1–2k слов/язык):** подмножество guess-словаря, из которого выбираются слова дня. Только общеизвестные конкретные существительные.

## 2. Построение guess-словаря

### Источники частотных списков
- RU: частотный словарь Ляшевской–Шарова (есть на GitHub) или частоты OpenCorpora.
- EN: библиотека `wordfreq` (`top_n_list('en', 100000)`) или SUBTLEX-US.

### Фильтры (порядок важен)
```python
for word in frequency_list:                     # уже отсортирован по частоте
    w = word.lower().replace('ё', 'е')
    if len(w) < 3 or len(w) > 15: continue      # исключение: ru допускает <3? нет — минимум 3
    if not w.isalpha(): continue                 # дефисы, цифры, апострофы — вон
    if w in blacklist: continue                  # мат, оскорбления (готовые списки RU/EN с GitHub)
    if is_proper_noun(w): continue               # см. ниже
    # Морфология:
    #  RU: pymorphy3 — оставляем NOUN, nomn, sing (плюс pluralia tantum: ножницы, брюки)
    #  EN: NLTK pos_tag / wordfreq + WordNet: оставляем NN (noun singular)
    if not is_valid_noun(w, lang): continue
    if w not in embedding_model: continue        # нет вектора — нет игры
    good.append(w)
    if len(good) >= 40000: break
```

**Имена собственные:** для RU — pymorphy3 теги `Name`, `Surn`, `Geox`, `Orgn`; для EN — WordNet: если слово встречается только как proper noun, выкинуть. Плюс ручной чёрный список стран/городов/брендов.

**Уменьшительные (RU):** желательно резать («котик», «котяра» рядом с «кот» бесят). Эвристика: если `normal_form` слова через pymorphy3 отличается от него самого при разборе как уменьшительного — оставить только базу. Практически: собрать топ-подозрительных суффиксов (-ик, -ок, -ёнок, -очка...) и проверить руками пограничные случаи. Не перфекционизм: достаточно вычистить то, что всплывёт в валидации (§5).

## 3. Answer-словарь

- Взять топ-2500 guess-словаря по частоте → **прогнать глазами** (~1 час): выкидывать абстракции («наличие», «отношение», «фактор»), канцелярит, всё, что «скучно угадывать». Цель — 1000–1500 слов. Это ручная работа, и именно она делает игру приятной.
- Перемешать с фиксированным seed → это и есть расписание слов дня (day_id → индекс). Записать в `daily_words`.

## 4. Генерация рангов

```
для каждого (day_id, lang):
    answer = daily_words[day_id, lang]
    sims = vectors @ vectors[answer]          # cosine (векторы заранее нормированы)
    ranks = argsort(-sims)                    # 1 = сам answer
    bulk insert (day_id, lang, word, rank) в words_rank
```
- Векторы: `float16`, L2-нормированные, порядок строк = `vocab_*.txt`. Эти же файлы едут на сервер для челленджей.
- 100 дней × 2 языка → ~10–20 минут генерации, БД 250–400 MB. `VACUUM` в конце.

## 5. Валидация глазами (обязательный gate)

Скрипт `validate.py <слово>` печатает топ-50 ближайших. Прогнать минимум по 10 слов на язык: вода, автомобиль, любовь, компьютер, собака, музыка, война, деньги, море, хлеб / water, car, love, computer, dog, music, war, money, sea, bread.

- **Плохо:** `компьютер → пк, комп, эвм, tft-монитор, юзер` (сленг, аббревиатуры) → добавить правило, пересобрать.
- **Хорошо:** `компьютер → ноутбук, программа, монитор, клавиатура, интернет, техника`.

Критерий готовности: в топ-30 любого проверочного слова нет мусора, который бы возмутил обычного игрока. Обычно 2–3 итерации фильтров.

Дополнительно: автоматический smoke-тест — для каждого слова answer-словаря проверить, что в его топ-10 нет слов из blacklist и слов, отличающихся от ответа только суффиксом.

## 6. Словарь EN-форм

Для каждого слова guess-словаря EN сгенерировать формы (lemminflect: множественное число; плюс исходное слово) → `en_forms.json {"apples":"apple", ...}`. Коллизии решать в пользу более частотной леммы.

## 7. Деплой данных

Скрипт `push_data.sh`: scp артефактов на VPS в volume `/data/` + `sqlite3` merge статических таблиц в рабочую БД (или сервер читает статические таблицы из отдельного файла `game_static.db` через `ATTACH` — предпочтительно: проще обновлять, runtime-данные не трогаются). Прогон при первом деплое и затем раз в ~3 месяца (долить дни).
